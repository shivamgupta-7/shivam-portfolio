Twitter Clone (Distributed Systems in Erlang) Click to expand
  • Project Objective: The aim was to implement a Twitter Clone and client tester/simulator using the Erlang programming language.
  • Twitter-Like Engine: The implementation included features such as user registration, sending tweets with hashtags and mentions, subscribing to users' tweets, re-tweets, and querying tweets based on subscriptions, hashtags, and mentions.
  • WebSocket Interface: The Cowboy web framework was used to create a WebSocket interface for real-time delivery of tweets. The design involved a JSON-based API, rewriting parts of the engine and client using WebSharper, and implementing public key-based authentication for secure communication.
  • Public Key Authentication: Users registered with a public key, and upon reconnecting via WebSockets, they authenticated using a challenge-based algorithm. The engine sent a challenge, the client signed it along with the current time, and the engine confirmed the authentication. Secret key establishment using Diffie-Helman protocol and HMAC signing of messages were also implemented for security.
